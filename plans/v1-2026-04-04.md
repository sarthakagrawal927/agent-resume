# claude-resume v1 — Implementation Plan

## Problem
Claude Code rate limits interrupt work. User walks away, wants it to auto-resume.

## Solution
Shell script that:
1. Detects Claude rate limits
2. Cascades to cheaper Claude models (Opus → Sonnet → Haiku)
3. Falls back to another AI agent via cli-continues if all Claude models are limited
4. Counts down + notifies when original model is available
5. Auto-resumes on original model

## Architecture
Single shell script (~200-300 lines), macOS-first, zero required deps.
Optional dep: `npx continues` for cross-agent fallback.

## Flow
```
User runs: claude-resume -c "continue the refactor"

1. Check rate limit: `claude -p 'check'`
2. If OK → run Claude normally, exit
3. If rate limited:
   a. Try Sonnet: `claude --model sonnet -p 'check'`
   b. If OK → run with Sonnet, done
   c. Try Haiku: `claude --model haiku -p 'check'`
   d. If OK → run with Haiku, done
   e. All Claude models limited:
      - If --fallback: export context via cli-continues, launch fallback tool
      - Start background countdown
      - Notify (macOS) when original model is available
      - User switches back manually
4. If --loop: repeat from 1
```

## CLI Interface
```
claude-resume [options] [prompt]

  -c, --continue           Continue previous conversation
  -f, --fallback <tool>    Fallback AI tool (gemini, codex, copilot, etc.)
  -m, --model <model>      Force specific model
  --cascade                Auto-cascade models on rate limit (default: on)
  --no-cascade             Disable model cascade
  --loop                   Keep retrying on repeated rate limits
  --skip-permissions       Use --dangerously-skip-permissions
  --test <seconds>         Simulate rate limit
  -h, --help               Show help
  -v, --version            Show version
```

## Key Design Decisions
- macOS-first: no GNU timeout dependency, BSD date everywhere
- No --dangerously-skip-permissions by default — opt-in via --skip-permissions
- cli-continues is optional — script works without it (just no cross-agent fallback)
- Model cascade is the primary strategy, cross-agent fallback is secondary
- Background notification, not auto-switching back (user controls when to return)
