Features

Everything you need to stay unblocked

One shell script handles rate limits, model switching, agent fallback, and task orchestration. You walk away, it keeps working.

⏱

Rate Limit Detection

Parses 3 known Claude output formats for rate limit timestamps. Live countdown timer with macOS native notifications when the limit lifts.

core
⇅

Model Cascading

When Opus is rate-limited, automatically try Sonnet, then Haiku. Your task continues on the first available model, transparently.

Opus → Sonnet → Haiku
🌐

Cross-Agent Fallback

All Claude models gone? Hand off to Gemini, Codex, or Copilot via cli-continues. Seamless context export and auto-resume when Claude returns.

gemini / codex / copilot
📤

Inter-Agent Delegation

Run tasks on other AIs headlessly and inject results as context into Claude. Gemini fetches data, Claude processes it.

delegate:agent
📋

Task Queues

Process markdown task files sequentially with full rate-limit resilience. Completed tasks are skipped on re-run. Walk away, come back to everything done.

- [ ] / - [x]
🔁

Loop Mode

Keep retrying through repeated rate limits until the task succeeds. Essential for long autonomous runs that hit limits multiple times.

--loop
📊

Status & Debug

See model check results, parsed timestamps, cascade decisions, and fallback attempts. Full visibility into what is happening under the hood.

DEBUG=1
🛡

Permissions Control

Skip-permissions is opt-in, not default. You explicitly choose the security tradeoff for unattended operation.

--skip-permissions

Installation

Up and running in 30 seconds

Clone, make executable, symlink. No build step, no dependencies, no package manager.

Requirements

  • ✓Claude Code CLI (claude)
  • ✓Bash 3+ (ships with macOS)

Optional

  • ○cli-continues — cross-agent fallback
  • ○Gemini CLI — delegation target
  • ○Codex CLI — delegation target
  • ○jq — debug output parsing
Install 
# Clone
git clone https://github.com/sarthakagrawal927/claude-resume.git
cd claude-resume

# Make executable
chmod +x claude-resume.sh

# Symlink to PATH
ln -s "$(pwd)/claude-resume.sh" /usr/local/bin/claude-resume
Verify 
claude-resume --version
claude-resume --help

Usage

Common workflows

Resume after rate limit

The simplest use case. Detects the rate limit, counts down, and resumes your conversation automatically. Model cascade kicks in if your primary model is still limited.

Terminal 
claude-resume -c

Long autonomous task

Loop mode retries through repeated rate limits. Skip-permissions enables unattended operation. Walk away and come back to a finished refactor.

Terminal 
claude-resume --loop --skip-permissions \
  -c "refactor all API endpoints to async/await"

Gemini fallback

When all Claude models are rate-limited, hand off context to Gemini. When Gemini finishes, automatically check if Claude is back and resume.

Terminal 
claude-resume -f gemini --loop -c

Delegate to another agent

Run a specific task on Gemini headlessly, capture the output, and inject it as context into Claude's next task. Perfect for accessing tools Claude cannot reach.

Terminal 
claude-resume delegate gemini \
  "query Stitch for user signups last 30 days"

Process a task queue

Feed a markdown file of tasks. Each is processed sequentially with full rate-limit handling. Completed tasks are skipped on re-run.

Terminal 
claude-resume tasks sprint.md --loop --skip-permissions

Test / simulation mode

Dry-run the countdown and resume flow without hitting real rate limits. Useful for verifying your setup works before a long run.

Terminal 
claude-resume --test 10 -c

How It Works

The cascade decision tree

When you run claude-resume --loop -f gemini -c, here is the full decision flow.

1

Check current model (Opus)

Test if your primary model is available by running a quick check. If it responds, run your task immediately.

Available? Run & done Rate limited? Next step
2

Model cascade

Try each cheaper model in order. Sonnet first, then Haiku. Your task continues on the first model that responds.

Sonnet OK? Run with Sonnet Haiku OK? Run with Haiku All limited? Next step
3

Cross-agent fallback

Hand your full context to Gemini (or Codex, Copilot) via cli-continues. Work continues on the fallback agent. When done, check if Claude is back.

Context exported to Gemini Claude back? Resume on Claude
4

Wait for rate limit reset

Parse the reset timestamp from Claude's output. Show a live HH:MM:SS countdown. Send a macOS notification when the limit lifts. Resume automatically.

Live countdown in terminal Limit lifted → resume
5

Loop (with --loop)

If rate limited again during execution, the entire cascade restarts from step 1. Keeps going until your task succeeds.

Rate limited again? Go to step 1
