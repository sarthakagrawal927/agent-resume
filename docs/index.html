Features

Everything you need to stay unblocked

One shell script handles rate limits, model switching, exhaustive agent fallback, sleep prevention, and task orchestration. You walk away, it keeps working.

⏱

Rate Limit Detection

Parses 6+ known Claude output formats for rate limit timestamps. Live countdown timer with macOS native notifications when the limit lifts.

core
⇅

Model Cascading

When Opus is rate-limited, automatically try Sonnet, then Haiku. Your task continues on the first available model, transparently.

Opus → Sonnet → Haiku
🌐

Exhaustive Agent Fallback

All Claude models gone? Automatically loops through every installed agent — Gemini, Codex, Copilot, Aider — until one succeeds. No manual configuration needed.

gemini / codex / copilot / aider
📤

Inter-Agent Delegation

Run tasks on other AIs headlessly and inject results as context into Claude. Gemini fetches data, Claude processes it.

delegate:agent
📋

Task Queues

Process markdown task files sequentially with full rate-limit resilience. Completed tasks are skipped on re-run. Walk away, come back to everything done.

- [ ] / - [x]
🔁

Loop Mode

Keep retrying through repeated rate limits until the task succeeds. Essential for long autonomous runs that hit limits multiple times.

--loop
🛡

Permission Modes

Choose your security level with --permission-mode: auto, acceptEdits, or bypassPermissions. Safer than the old all-or-nothing skip-permissions flag.

--permission-mode auto
➡

Flag Pass-Through

Forward any extra flags directly to the Claude CLI using --. Pass worktree options, verbose flags, or anything else Claude supports.

-- <flags>
☕

Sleep Prevention

Automatically runs caffeinate -dims on macOS during loop mode and task processing. Your Mac stays awake through long-running operations.

caffeinate
🧹

Signal Handling

Proper cleanup on exit via trap handlers. Kills caffeinate, logs session end, and preserves exit codes on SIGINT, SIGTERM, and normal exit.

trap cleanup EXIT
📊

Status & Debug

See model check results, parsed timestamps, cascade decisions, and fallback attempts. Full visibility into what is happening under the hood.

DEBUG=1
📝

Event Logging

Log every rate limit hit, cascade decision, agent fallback, and session boundary to a file. Trace exactly what happened during long autonomous runs.

--log events.log

Installation

Up and running in 30 seconds

Clone, make executable, symlink. No build step, no dependencies, no package manager.

Requirements

  • ✓Claude Code CLI (claude)
  • ✓Bash 3+ (ships with macOS)

Optional

  • ○Gemini CLI — fallback / delegation target
  • ○Codex CLI — fallback / delegation target
  • ○Copilot CLI — fallback / delegation target
  • ○Aider — fallback / delegation target
  • ○cli-continues — interactive cross-agent handoff
Install 
# Clone
git clone https://github.com/sarthakagrawal927/claude-resume.git
cd claude-resume

# Make executable
chmod +x claude-resume.sh

# Symlink to PATH
ln -s "$(pwd)/claude-resume.sh" /usr/local/bin/claude-resume
Verify 
claude-resume --version
claude-resume --help

Usage

Common workflows

Resume after rate limit

The simplest use case. Detects the rate limit, counts down, and resumes your conversation automatically. Model cascade kicks in if your primary model is still limited.

Terminal 
claude-resume -c

Long autonomous task

Loop mode retries through repeated rate limits. Permission mode controls security level for unattended operation. Sleep prevention keeps your Mac awake automatically.

Terminal 
claude-resume --loop --permission-mode auto \
  -c "refactor all API endpoints to async/await"

Exhaustive agent fallback

When all Claude models are rate-limited, automatically tries every installed agent (Gemini, Codex, Copilot, Aider) in order. Use -f to prioritize a specific agent first.

Terminal 
# Auto-detect all installed agents
claude-resume --loop -c

# Prioritize Gemini, then try others
claude-resume -f gemini --loop -c

Delegate to another agent

Run a specific task on Gemini headlessly, capture the output, and inject it as context into Claude's next task. Perfect for accessing tools Claude cannot reach.

Terminal 
claude-resume delegate gemini \
  "query Stitch for user signups last 30 days"

Process a task queue

Feed a markdown file of tasks. Each is processed sequentially with full rate-limit handling. Completed tasks are skipped on re-run.

Terminal 
claude-resume tasks sprint.md --loop \
  --permission-mode acceptEdits

Flag pass-through

Forward extra flags directly to the Claude CLI. Everything after -- is passed through untouched. Useful for worktrees, verbose mode, or any future Claude flags.

Terminal 
claude-resume -c -- --worktree myfeature

Test / simulation mode

Dry-run the countdown and resume flow without hitting real rate limits. Useful for verifying your setup works before a long run.

Terminal 
claude-resume --test 10 -c

How It Works

The cascade decision tree

When you run claude-resume --loop -c, here is the full decision flow: Claude Models → All Installed Agents → Wait → Retry.

1

Check current model (Opus)

Test if your primary model is available by running a quick check. If it responds, run your task immediately. Sleep prevention activates automatically in loop mode.

Available? Run & done Rate limited? Next step
2

Claude model cascade

Try each cheaper model in order. Sonnet first, then Haiku. Your task continues on the first model that responds.

Sonnet OK? Run with Sonnet Haiku OK? Run with Haiku All limited? Next step
3

All installed agents (exhaustive)

Loop through every installed agent automatically: Gemini, Codex, Copilot, Aider. Each is tried in order. If one succeeds, the task is done. If -f is set, that agent is tried first.

gemini → codex → copilot → aider Any agent succeeds? Done All exhausted? Next step
4

Wait for rate limit reset

Parse the reset timestamp from Claude's output. Show a live HH:MM:SS countdown. Send a macOS notification when the limit lifts. Resume automatically.

Live countdown in terminal Limit lifted → resume
5

Retry (with --loop)

If rate limited again during execution, the entire cascade restarts from step 1. Keeps going until your task succeeds. Signal handlers ensure clean shutdown if you interrupt.

Rate limited again? Go to step 1
