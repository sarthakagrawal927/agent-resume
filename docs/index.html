Features

Everything you need to stay unblocked

One shell script handles smart routing, unified cascading, per-agent rate limit detection, sleep prevention, and task orchestration across every major AI coding agent.

🧭

Smart Routing

Probes all installed agents on startup and routes your task to the best available one by quality tier. No configuration needed — it just picks the best agent that is not rate-limited.

default behavior
⇅

Unified Cascade

One cascade across ALL agents, ordered by quality tier. When your primary agent hits a rate limit mid-task, it automatically cascades to the next available agent in the registry.

T1 → T2 → T3
🎯

Quality Tiers

Agents are ranked by quality: T1 (Claude Opus, Gemini), T2 (Claude Sonnet, Codex), T3 (Claude Haiku, Copilot, Aider). The best available tier always runs first.

7 agents across 3 tiers
🔍

Per-Agent Rate Limit Detection

Each agent has its own rate limit patterns. Gemini: RESOURCE_EXHAUSTED/429. Codex: exceeded retry limit. Copilot: rate_limited. Aider: RateLimitError. All detected automatically.

agent-specific patterns
🔧

Custom Ordering

Override the default tier order with --order gemini,claude-opus,codex. Define your own cascade priority to match your workflow and subscription tiers.

--order a,b,c
🔁

Loop Mode

Keep retrying through ALL agents until the task succeeds. Essential for long autonomous runs that hit multiple rate limits across different providers.

--loop
📤

Inter-Agent Delegation

Run tasks on specific agents headlessly and inject results as context into the next task. Gemini fetches data, Claude processes it — all orchestrated automatically.

delegate:agent
📋

Task Queues

Process markdown task files sequentially with full rate-limit resilience and cross-agent delegation. Completed tasks are skipped on re-run. Walk away, come back to everything done.

- [ ] / - [x]
🛡

Permission Modes

Choose your security level with --permission-mode: auto, acceptEdits, or bypassPermissions. Fine-grained control for unattended operation.

--permission-mode auto
☕

Sleep Prevention

Automatically runs caffeinate -dims on macOS during loop mode and task processing. Your Mac stays awake through long-running operations.

caffeinate
📊

Status Dashboard

Run agent-resume status to see every agent, its tier, installation status, and whether it is available or rate-limited right now.

agent-resume status
📝

Event Logging

Log every probe, route decision, rate limit hit, cascade step, and session boundary to a file. Full audit trail for long autonomous runs.

--log events.log

Installation

Up and running in 30 seconds

Clone, make executable, symlink. No build step, no dependencies, no package manager.

Requirements

  • ✓Bash 3+ (ships with macOS)
  • ✓At least one AI agent CLI installed

Supported Agents

  • T1Claude Opus (claude), Gemini (gemini)
  • T2Claude Sonnet (claude), Codex (codex)
  • T3Claude Haiku (claude), Copilot (copilot), Aider (aider)

Optional

  • ○cli-continues — interactive cross-agent handoff
Install 
# Clone
git clone https://github.com/sarthakagrawal927/agent-resume.git
cd agent-resume

# Make executable
chmod +x agent-resume.sh

# Symlink to PATH
ln -s "$(pwd)/agent-resume.sh" /usr/local/bin/agent-resume
Verify 
agent-resume --version
agent-resume status

Usage

Common workflows

Smart route to best agent

The default behavior. Probes all installed agents, routes to the best available one by quality tier, and cascades to the next if it gets rate-limited mid-task.

Terminal 
agent-resume -c

Long autonomous task

Loop mode retries through all agents on repeated rate limits. Permission mode controls security level for unattended operation. Sleep prevention keeps your Mac awake.

Terminal 
agent-resume --loop --permission-mode auto \
  -c "refactor all API endpoints to async/await"

Custom cascade order

Override the default tier-based ordering. Set your preferred cascade priority to match your subscriptions and workflow. Only agents you list are tried.

Terminal 
# Gemini first, then Opus, then Codex
agent-resume --order gemini,claude-opus,codex \
  --loop -c "implement search feature"

Single agent, no cascade

Disable cascading to force a single agent. If it is rate-limited, waits for the reset instead of falling through to other agents.

Terminal 
agent-resume --no-cascade -m opus -c

Delegate to another agent

Run a specific task on any agent headlessly, capture the output, and inject it as context into the next task. Access tools one agent cannot reach through another.

Terminal 
agent-resume delegate gemini \
  "query Stitch for user signups last 30 days"

Process a task queue

Feed a markdown file of tasks. Each is processed sequentially with full rate-limit handling and cross-agent cascade. Completed tasks are skipped on re-run.

Terminal 
agent-resume tasks sprint.md --loop \
  --permission-mode acceptEdits

Check all agent statuses

See every agent in the registry with its tier, installation status, and current availability. Quickly identify which agents are ready and which are rate-limited.

Terminal 
agent-resume status

How It Works

Unified smart routing + cascade

When you run agent-resume --loop -c, the script probes all installed agents and routes to the best one. If rate-limited, it cascades down the unified tier order.

1

Probe all installed agents

On startup, agent-resume probes every installed agent in tier order. Each agent is checked with its own probe command to determine availability. The first one that responds without a rate limit error is selected.

T1: Claude Opus, Gemini T2: Sonnet, Codex T3: Haiku, Copilot, Aider
2

Route to best available agent

Your task runs on the highest-tier agent that is not rate-limited. Each agent's rate limit is detected with agent-specific patterns — Gemini looks for RESOURCE_EXHAUSTED, Codex for 429 retry limits, Copilot for rate_limited, Aider for RateLimitError.

Available? Run task immediately Rate limited mid-task? Cascade
3

Cascade to next agent

If the running agent hits a rate limit during execution, the task automatically moves to the next agent in the cascade. No separate "model cascade" vs "agent fallback" — one unified registry handles everything.

claude-opus → gemini → claude-sonnet → codex → ... Any agent succeeds? Done
4

All exhausted? Wait for reset

When every installed agent is rate-limited, parse the reset timestamp and show a live HH:MM:SS countdown. macOS notification fires when the limit lifts. Resume automatically on the first agent.

Live countdown in terminal Limit lifted → resume
5

Retry (with --loop)

If rate-limited again during execution, the entire routing + cascade restarts from step 1. With --order, your custom priority is used instead of the default tier order. Signal handlers ensure clean shutdown on interrupt.

Rate limited again? Back to step 1
